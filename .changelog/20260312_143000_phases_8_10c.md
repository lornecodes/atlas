# Phases 8-10C: Security, Skills, MCP Federation

**Date**: 2026-03-12

## Phase 8: Security & Sandboxing
- `atlas/security/policy.py` — SecurityPolicy from YAML (permission scopes, secret rules)
- `atlas/security/secrets.py` — SecretResolver with EnvSecretProvider and FileSecretProvider
- `atlas/contract/permissions.py` — PermissionsSpec (file, network, subprocess, env scopes)
- Permission enforcement in ExecutionPool before agent execution
- CLI: `--security-policy` flag on serve command
- 60+ security tests

## Phase 9: Skills & Tool Use
- `atlas/skills/` — SkillRegistry, SkillResolver, SkillSpec, skill.yaml schema
- `atlas/skills/platform.py` — PlatformToolProvider with 12 `atlas.*` platform tools
- `requires.skills` in agent contracts, resolved and injected at execution time
- `AgentContext.skill()` — agents call tools by name
- CLI: `skill list`, `skill inspect` commands
- 40+ skill tests

## Phase 10A: MCP Server (Network Transport + Auth)
- `atlas/mcp/server.py` — `create_mcp_server()` wraps SkillRegistry as MCP tool server
- `atlas/mcp/transport.py` — ASGI app with Streamable HTTP + SSE + health endpoint
- `atlas/mcp/auth.py` — BearerAuthMiddleware (timing-safe token validation, /health exempt)
- CLI: `atlas mcp --serve --port 8400 --auth-token secret`
- CLI: `atlas serve --mcp-port 8400` (MCP alongside REST)
- 19 transport/auth tests

## Phase 10B: MCP Client (Remote Tool Federation)
- `atlas/mcp/client.py` — RemoteToolProvider connects to remote MCP servers
- Discovers tools, registers as namespaced local skills (`{name}.{tool}`)
- `parse_remote_spec()` — CLI helper for `name=url@token` format
- CLI: `--remote` flag on serve and mcp commands
- 24 client tests

## Phase 10C: Federated Chains (Remote Agent Execution)
- `atlas/mcp/remote_agents.py` — RemoteAgentProvider + RemoteAgent
- `atlas.exec.run` platform tool — synchronous agent execution (federation primitive)
- `AgentRegistry.register_virtual()` + `unregister()` — programmatic agent registration
- ChainRunner skill injection — optional SkillResolver resolves `requires.skills` per step
- Virtual agents: chains reference `lab.translator` directly, executes on remote instance
- 34 remote agent tests
